# Implementation Plan: Event Bridge

**Branch**: `001-event-bridge` | **Date**: 2026-02-25 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/001-event-bridge/spec.md`

## Summary

Build agent-pulse, a Go CLI tool that bridges Claude Code lifecycle events to physical hardware. The system has two commands: `setup` (configures Claude Code hooks in a project) and `serve` (runs an HTTP server that receives events and broadcasts them to WebSocket-connected devices). The server auto-starts via hooks, requires zero manual maintenance, and supports device-agnostic connections.

## Technical Context

**Language/Version**: Go 1.24+
**Primary Dependencies**: cobra (CLI), chi (HTTP router), gorilla/websocket (WebSocket), slog (structured logging), gopkg.in/yaml.v3 (config)
**Storage**: YAML config file at `~/.config/agent-pulse/config.yaml` (no database)
**Testing**: Go table-driven tests, `httptest` for HTTP handlers, gorilla test utilities for WebSocket
**Target Platform**: macOS (primary), Linux (secondary). Single binary distribution.
**Project Type**: CLI tool + background server
**Performance Goals**: Event delivery <1s latency, 10+ concurrent WebSocket clients
**Constraints**: Minimal resource usage (background process), single port, no external service dependencies
**Scale/Scope**: Single-user, single-machine, local network

```text
└── setup.go             # setup subcommand — writes hooks to settings.json

internal/
├── server/
│   ├── server.go        # HTTP server setup, chi router, graceful shutdown
│   ├── handler.go       # POST /event handler
│   └── websocket.go     # WebSocket upgrade, Hub, Client, readPump, writePump
├── hooks/
│   └── setup.go         # Read/merge/write .claude/settings.json

```

**Structure Decision**: Standard Go CLI layout from the technical spec. `cmd/` for cobra commands, `internal/` for business logic split by domain (server, hooks, config). No `pkg/` directory — everything is internal. Tests live alongside source files (`*_test.go`).
